# Plan: OpenRouter Quote Generator

## Current State

- GitHub Actions workflow fetches quotes from a public API (`programming-quotesapi.vercel.app`)
- Runs on schedule: every 12-16-19 hours
- Updates `QUOTES.md` with markdown table entries: timestamp, author, quote
- Automatically commits and pushes changes
- Supports manual triggering via `workflow_dispatch`

## Goal

Replace static API with OpenRouter LLM to generate contextual, motivational quotes matching a specific ethos:
- **Technical truth** — insights only engineers appreciate
- **Emotional resonance** — validation, perseverance, capability when stuck
- **Brevity** — under 20 words, quotable
- Example: "The complexity you're seeing is proof you understand the problem. Now trust yourself to solve it."

## Implementation Steps

### 1. Set Up OpenRouter Integration

**File:** `.github/workflows/master.yml`

- Add `OPENROUTER_API_KEY` as a GitHub secret
- Replace the curl API call (step: "Fetch Quote and Update Table") with an OpenRouter call
- Use Python or bash script to call OpenRouter, parse response, and update `QUOTES.md`

**Decision points:**
- Model choice: `openai/gpt-4o-mini` (recommended for speed/cost) or alternative?
- Author field: "OpenRouter LLM" or custom name?

### 2. Create Prompt Template

**Location:** Within workflow step or separate file

**Prompt constraints:**
- Generate ONE motivational quote for software engineers
- Under 20 words
- No clichés, no motivational fluff, no hustle culture
- Focus: debugging, architecture, reliability, learning, technical debt, teamwork, failure, craftsmanship
- Tone: reflective, witty, slightly wise
- Sound like a senior engineer's hard-earned insight
- Be validating when someone is stuck: "you can solve this"

**Example outputs to match:**
- "The complexity you're seeing is proof you understand the problem. Now trust yourself to solve it."
- "Every system you shipped once seemed impossible."
- "Stuck is where breakthroughs live."

### 3. Modify Workflow Step

**Current step:** Fetches from static API, parses JSON with `jq`

**New step:** Call OpenRouter API, format response, update table

Workflow remains the same:
- Table initialization (if needed)
- Append new row: `| timestamp | author | quote |`
- Update `TIMESTAMP.txt` for backwards compatibility
- Git commit with random emoji message
- Push to main

### 4. Testing Strategy

- **Local test first** (optional): Script to validate prompt and API call
- **Workflow validation**: First scheduled run or manual `workflow_dispatch` to verify formatting

## Deliverables

1. Updated `.github/workflows/master.yml` with OpenRouter integration
2. Prompt template (inline in workflow or documented separately)
3. Validated `QUOTES.md` entries with new quote format

## Dependencies

- OpenRouter API key (set as GitHub secret)
- Model availability (verify chosen model is available via OpenRouter)
- Bash/Python for API call and JSON parsing

## Success Criteria

- Quote generated by OpenRouter appends to `QUOTES.md` successfully
- Markdown table formatting is valid
- Commit is created and pushed automatically
- Quote matches ethos: technical, validating, concise, no clichés
